# Worklog: FLUX.1 Curation

**Started**: 2026-03-15 ~07:00 UTC
**Generator**: FLUX.1 [pro], [dev], [schnell], [redux], [kontext (annotated)] (Black Forest Labs)
**Target**: 50-100K validated photorealistic images (soft target)
**Exclusions**: FLUX.1 Fill (mixes real content). FLUX.2 excluded.
**Content policy**: Photorealistic only. Keep NSFW except exposed genitals.
**Reddit**: Avoided — Civitai volume is sufficient.

- Variants: pro, dev, schnell, redux, kontext (annotated only)

- No content policy beyond photorealistic filter
- FSD: no longer needs --weights-dir flag
- User later updated target to 50-100K

## Phase 1: Source Research

5 research agents launched in parallel:

- Model ID: 618692 ("FLUX" by Black Forest Labs)

- Pro 1.1 Ultra (1088507): ~1.6M gens, ~4% on-site, high-res

- Kontext Max (1892523): 12 items total

### Higgsfield (small, Kontext only)
- Only flux_kontext (94 posts) and flux_2 (304 posts)
- No FLUX.1 base models at all
- flux_kontext has input_images (real content mixing) — annotated as flux_kontext

### Yodayo (moderate, LoRA contamination)
- FLUX.1 [dev]: version UUID d5d0dae7-93e5-47a0-9d45-79687078db65, ~4,400 accessible
- FLUX.1 [schnell]: version UUID be2fa7fb-aa53-4b9d-8f2f-72e41bf63762, ~750 accessible

### AIGCArena (not viable)
- No FLUX.1 — only FLUX.2 [max]. Skipped.

### Tensor.Art (moderate, pagination capped)
- 1.1M FLUX-tagged posts — massive but pagination caps at ~1,000 posts
- Built scraper with Playwright + HMAC signing header capture
- Filters for FLUX.1 base model only (not FLUX.2, SD, etc.)
- Pagination limit: ~1,000 posts regardless of sort order
- Net yield: ~3,092 unique images

### Other Platforms Researched

- **OpenArt.ai**: Public API discovered (no Playwright needed). Exhausted at 125 FLUX items.
- **Krea.ai, Leonardo.ai, NightCafe**: Viable but not needed given Civitai volume.
- **fal.ai, Replicate, Pollinations**: Generation-only, no galleries.

## Phase 3: Validation (ongoing)

- Sweep loop running: batch_classify → pipeline → sleep 120s → repeat
- ~47% content rejection rate (digital art, anime, illustration dominant)
- ~165 too_large rejections (12-25MP upscaled images from Tensor.Art)
- Added "digital drawing" to REJECT_KEYWORDS (1 leaker found in spot check)
- Spot-check confirmed: accepted images are photorealistic, rejected are correct
- **~4,657 validated** as of 10:07 UTC

### Content Rejection Breakdown (before "digital art" fix)
- digital art: 7,392 (removed — too generic, 40.8% false positive rate)
- digital painting: 1,672
- illustration: 1,565
- anime: 1,540
- cgi: 638
- Other: watercolor, comic, sketch, cartoon, speech bubble, etc.

### "digital art" Keyword Fix
- Discovered JoyCaption uses "digital art" as style qualifier on photorealistic images
- Captions like "photograph, digital art, realistic style" were false positives
- Removed from all generator configs (flux1, seedream4, gpt_image_1, nano_banana_1_2)
- Recovered: 3,459 FLUX + 1,413 ChatGPT + 453 Seedream + 733 Nano Banana = 6,058 total
- Pass rate improved from ~40% to ~62%

## Phase 4: Finalization

### Civitai Async Deadlock Fix
- async producer/consumer scraper deadlocked after ~100-400 downloads
- Root cause: threads blocked on slow HTTP in asyncio.to_thread, pool exhaustion
- Built civitai_simple.py — synchronous, never hangs
- Ran 3 instances in parallel (Dev, Schnell, Krea) for ~2 img/s combined

### FSD Detection
- **81.5% detection rate** (18,997/23,296)
- Mean z-score: -7.07, Median: -6.49
- Higher than Seedream (11.7%) despite both being DiT architectures
- FLUX uses MMDiT (rectified flow) vs Seedream's DiT+MoE
